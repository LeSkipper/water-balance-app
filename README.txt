# Water Balance 💧 - Daily Hydration Tracker

**Water Balance** — это кроссплатформенное приложение на Flutter, предназначенное для ежедневного отслеживания уровня гидратации, контроля количества выпитой воды и формирования полезной привычки.

## 🌟 Ключевые возможности

- **Аутентификация и Профили:** Собственная учетная запись для каждого пользователя с сохранением персональных параметров (вес, рост, возраст) для корректировки индивидуальной нормы воды. Безопасная аутентификация с хешированием паролей.
- **Отслеживание выпитой воды:** Удобное добавление порций воды (Quick Add) с автоматическим сохранением точного времени и даты.
- **Визуализация прогресса:** 
  - Наглядный ежедневный круговой прогресс-бар достижения цели (WaterProgress).
  - Графики потребления воды по дням недели (WeeklyChart).
  - Подсчет текущей серии дней подряд (Streaks), в которые была достигнута ежедневная цель.
- **Гибкие настройки профиля:**
  - Индивидуальная ежедневная цель (по умолчанию 2000 мл) с возможностью ручного изменения.
  - Настройка времени пробуждения и сна.
  - Выбор системы мер (мл, унции) и темы оформления (Светлая, Тёмная, Авто/Системная).
- **Мультиязычность:** Поддержка 5 языков (Английский, Русский, Французский, Немецкий, Испанский) из коробки с мгновенной сменой без перезагрузки.
- **Локальная база данных:** Безопасное хранение всех данных в SQLite (через `sqflite`), работающее в оффлайн-режиме. Поддержка десктопных ОС через `sqflite_common_ffi`.

## 🛠 Технологический стек

- **Фреймворк:** [Flutter](https://flutter.dev/) (SDK >=3.0.0 <4.0.0)
- **Язык:** Dart
- **Локализация:** `flutter_localizations`, `intl` (файлы формата `.arb`)
- **Управление состоянием:** `provider`
- **Маршрутизация:** `go_router` (декларативный роутинг)
- **Хранение данных:** 
  - `sqflite` (iOS / Android)
  - `sqflite_common_ffi` (Windows / Linux / macOS)
- **Утилиты:** `uuid` (генерация ID логов), `crypto` (sha256 хеширование), `google_fonts`

## ⚙️ Техническая информация в деталях

### 1. Управление состоянием (State Management)
Проект использует паттерн **Single Source of Truth** благодаря связке паттерна Singleton на уровне БД (`DatabaseHelper.instance`) и глобального провайдера `AppProvider` (наследуется от `ChangeNotifier`). 
- **`AppProvider`** кэширует данные текущего пользователя в оперативной памяти: `_user`, `_settings`, `_entries` (логи за сегодня), `_weeklyData` (агрегированная статистика) и `_streak`.
- Изменения состояния уведомляют слушателей через функцию `notifyListeners()`.
- Данные реактивно обновляются в UI посредством `context.watch<AppProvider>()` (или `context.read` для разового чтения/вызова методов).

### 2. Маршрутизация (Routing / Navigation)
Используется пакет **`go_router`**, поддерживающий Web и Desktop навигацию на основе URL. 
- Навигация построена вокруг `ShellRoute` — этот слой оборачивает основные экраны (`/`, `/stats`, `/profile`, `/settings`) в общий каркас, содержащий нижнее меню навигации (`BottomNavigationBar`).
- **Гарды (Auth Redirects):** На уровне реализации роутера настроена проверка авторизации (коллбэк `redirect`). Если пользователь не авторизован и пытается зайти на закрытый маршрут, система автоматически направляет его на экран `/login`.

### 3. Структура Базы Данных SQLite (Глубокий разбор)
В приложении реализована реляционная база данных (через `sqflite` и `sqflite_common_ffi`), состоящая из трёх связанных таблиц:
1. **`users`**:
   - Хранит аккаунты. Поля: `id` (PK), `name`, `email` (UNIQUE), `password_hash` (SHA-256 хэш, сформированный через пакет `crypto`), `weight`, `height`, `age`, `gender`, `join_date`.
2. **`settings`**:
   - Хранит индивидуальные интерфейсные и функциональные настройки. Поля: `id` (PK), `user_id` (FK), `goal` (цель потребления воды), `unit` (ml/oz), `wake_up_time`, `bed_time`, `theme` и `language`.
3. **`intake_entries`**:
   - Лог выпитой воды. Поля: `id` (PK, генерируемая строка UUID), `user_id` (FK), `amount` (Объем в мл), `time`, `date`.
   - Запросы к статистике используют агрегацию, например `COALESCE(SUM(amount), 0)`.

### 4. Дизайн и Темизация (Theming Framework)
Имплементирована кастомная система цветов через механизм **`ThemeExtension`**, которая реализована в классе `AppColorsData`.
- Расширение встраивается в базовый класс `ThemeData` (через поле `extensions`).
- Доступ в верстке осуществляется через элегантный экстеншен для BuildContext, например: `context.colors.textMuted` или `context.colors.bgGradient`.
- Вся палитра разделена на 2 константы: Светлую (`light`) и Темную (`dark`) версии с поддержкой прозрачностей, градиентов и семантических цветов (success, danger, warning).

### 5. Локализация (i18n)
Полностью поддержан стандарт локализации через файлы `.arb` в папке `lib/l10n`.
- При сборке (благодаря флагу `generate: true` в pubspec) генерируются классы локализаций. В коде доступ осуществляется так: `AppLocalizations.of(context)!.stayHydrated`. 
- Смена языка пользователем вызывает изменение геттера `locale` в провайдере, который реактивно обновляет параметр `locale` в корне виджета `MaterialApp.router`, меняя интерфейс "на лету" без перезапуска приложения и без потери состояния.

## 📂 Архитектура и структура проекта

Проект организован по современным стандартам:
```text
lib/
 ├── l10n/              # Файлы локализации (.arb)
 ├── models/            # Модели данных (UserProfile, AppSettings, IntakeEntry)
 ├── pages/             # Основные экраны (HomePage, LoginPage, SettingsPage и др.)
 ├── providers/         # Бизнес-логика (AppProvider)
 ├── router/            # Конфигурация маршрутов (AppRouter)
 ├── services/          # Сервисы доступа к данным (DatabaseHelper)
 ├── theme/             # Управление стилями, цветами (AppTheme / AppColorsData)
 ├── widgets/           # UI-компоненты (WaterProgress, WeeklyChart и др.)
 └── main.dart          # Точка входа
```

## 🚀 Установка и запуск

1. **Клонируйте репозиторий:**
   ```bash
   git clone <URL_репозитория>
   cd water_balance_flutter
   ```

2. **Загрузите зависимости:**
   ```bash
   flutter pub get
   ```

3. **Сгенерируйте ресурсы (localizations):**
   ```bash
   flutter gen-l10n
   ```

4. **Запустите проект:**
   - Для мобильных устройств (iOS/Android):
     ```bash
     flutter run
     ```
   - Для десктопных платформ (Windows/Linux) FFI активируется автоматически в `main.dart`:
     ```bash
     flutter run -d windows
     ```
